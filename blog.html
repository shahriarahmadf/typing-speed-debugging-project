Blog: Important Questions


1. What are the differences between localStorage and sessionStorage?
When some data is stored in localStorage, it remains. On the other hand, when data is stored in sessionStorage, it does not remain stored after the browser is closed. It is only available during the browser session. But the data in the localStorage can be accessed after closing the current browser session and opening it again later at any time. localStorage can save more data in size than sessionStorage.

2. What are the differences between global scope and block scope?
Under global scope, the variable is declared outside any block (of a loop or conditional statement) and is accessible in everywhere of the program. Under block scope, the variable is declared inside a block (of a loop or conditional statement) and can only be accessed under that block.

3. How does JavaScript event loop work?
The event loop is a process that waits for the Call Stack to be clear before pushing callbacks from the Task Queue to the Call Stack. Once the Stack is clear, the event loop triggers and checks the Task Queue for available callbacks. If there are any, it pushes it to the Call Stack, waits for the Call Stack to be clear again, and repeats the same process. (Source: geekflare.com)

4. In how many ways can we get undefined?
There are 8 ways to get undefined:

  1. If a variable is not initialized before usage
  2. If a function has no return statement, the return value will be undefined
  3. If a parameter of a function is not passed into it, the parameter inside the function will be undefined
  4. If return inside a function has nothing on the right side: return ; like this, then it will return undefined
  5. A property that does not exist in an object will give undefined
  6. If an array element is accessed outside of its actual index range
  7. If an element is deleted inside an array
  8. If the value is directly set to undefined
